Typography rules

Universal rules
  • Use clamp(min, vw, max) for all display and H1–H2 sizes. H3 and below can be fixed px — they're stable enough at all viewports.
  • Manrope headings always carry negative letter-spacing: -0.025em at hero, tapering to -0.005em by H4. Never use zero or positive tracking on headings.
  • IBM Plex Mono labels always use +0.08em to +0.12em letter-spacing + text-transform: uppercase. Required for readability at 9–11px.
  • Body line-height is always 1.7. Use 1.4 only for buttons and labels. Never below 1.4.
  • Teal accent #0FA3B1 is used for inline heading spans, CTAs, and label color. Gold is reserved for "featured / achievement" badges only.
  • On mobile, increase line-height by 0.05 (e.g. body goes 1.7 → 1.75) and ensure minimum tap targets of 44px for any clickable text.
  • Never mix fonts within a role. Headings = Manrope only. Body = Inter only. Labels/Logo = IBM Plex Mono only. No exceptions in production.
  • Never use unloaded weights. Loaded weights are: IBM Plex Mono 400/600/700, Manrope 400/600/700/800, Inter 400/500/600. Using 300 or 900 triggers FOUT.
  • Never center-align body paragraphs. Center only: hero H1, section micro-labels, short CTAs. All paragraphs are left-aligned.
  • Gold #D4AF37 fails WCAG AA on light backgrounds. On light mode use deep gold #B8860B for text. Gold is safe only on dark surfaces or as a background fill color.
  • On mobile, never set body text below 14px. Labels are the only exception at 10px (short, uppercase, high contrast).
Responsive behavior cheatsheet

Mobile <768px

  • Stack nav: hamburger menu, logo centered
  • Hero H1: clamp(26px,8vw,32px)
  • Body min: 14px
  • Project cards: single column
  • CTA buttons: full width w-100

Tablet 768–1199px

  • Nav: full links visible, compact
  • Hero H1: clamp(32px,5vw,40px)
  • Body: 15–16px
  • Project cards: 2-column grid
  • CTA buttons: inline

Desktop 1200px+

Hero H1: 48–64px
Body: 16px
Cards: 2–3 col grid
Nav logo: 16–18px
Section H2: 36px
Hero sub: 18px
